Database Setup

Prerequisites

  • PostgreSQL 12 or higher
  • Node.js 18 or higher
  • pnpm package manager

Installation Steps

  1. Clone the repository
  2. Install dependencies: pnpm install
  3. Create PostgreSQL database: createdb community_watch_db
  4. Run schema: psql -d community_watch_db -f database/schema.sql
  5. Start development server: pnpm dev

Database Schema

The system uses 7 main database tables:

  • CITIZEN - Stores community member information
  • STAFF - Stores authority and admin accounts
  • INCIDENT_TYPE - Predefined incident categories
  • REPORT - Crime and incident reports
  • REPORT_MEDIA - Media evidence (photos/videos)
  • AUTHORITY_ACTION - Actions taken by authorities
  • REPORT_RESOLUTION - Final resolution status

See schema.sql for complete database structure.

Validation Rules

Password Requirements

  • Minimum 8 characters
  • At least one uppercase letter
  • At least one lowercase letter
  • At least one number
  • At least one special character (!@#$%^&*)

IC Number Format

Malaysian IC format: YYMMDD-PB-####

Example: 920101-14-5678

Phone Number Format

Malaysian mobile format: 01XXXXXXXXX

Example: 0123456789 (10-11 digits)

Incident Types

  • Theft - Property theft or burglary
  • Vandalism - Property damage or graffiti
  • Suspicious Activity - Unusual or concerning behavior
  • Assault - Physical violence or threat
  • Burglary - Breaking and entering
  • Drug Activity - Suspected drug-related incidents
  • Others - Custom incident type
